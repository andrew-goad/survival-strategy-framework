## Source Implementation

### `src/survival_strategy_framework.py`

The main implementation contains the complete analytical workflow:

```text
Synthetic Retention Demonstration Data
→ Input Contract Validation
→ Persona Discovery
→ Regularized CoxPH Model
→ Stratified K-Fold Validation
→ Out-of-Fold Calibration
→ Proportional-Hazards Diagnostics
→ Model-Derived Risk Stratification
→ Same-Cohort Scenario Simulation
→ Dependency Synchronization
→ Predicted Survival Comparison
→ Run Registry
→ Executive PPTX
→ Technical PDF
```

Key implementation capabilities include:

- deterministic synthetic data generation
- binary event and right-censoring support
- governed duration, event, ID, feature, and scenario contracts
- K-Means persona discovery
- regularized Cox proportional hazards modeling
- train-only feature scaling within cross-validation folds
- out-of-fold discrimination and calibration
- raw and multiplicity-adjusted PH diagnostics
- scaled Schoenfeld residual evidence
- PH sensitivity analysis
- percentile-based risk stratification
- same-cohort control, improvement, and stress scenarios
- interaction and squared-term dependency rebuilding
- configured-versus-realized scenario-change audit
- model-predicted survival comparison
- deterministic substantive reproducibility checks
- automated stakeholder evidence generation

## Sample Data Policy

The repository uses synthetic demonstration data only.

The uploaded `input_snapshot.csv` is included to make the validated run inspectable and reproducible. It contains no real customer records, PII, proprietary business rules, booked-account histories, or production treatment strategies.

The output files represent:

- synthetic time-to-event observations
- model-development and validation evidence
- controlled same-cohort scenario sensitivities
- generated executive and technical reporting
- run-level governance artifacts

They do not represent production retention forecasts or causal intervention results.

## Interpretation Boundaries

The project is validated as a governed analytical demonstration—not as a production retention system.

Important boundaries:

- The data is synthetic and contains no PII.
- Partial hazard is relative modeled risk, not event probability.
- Kaplan-Meier curves are descriptive observed evidence.
- CoxPH-predicted survival is model-based and requires domain validation before production use.
- Scenario movement is controlled sensitivity, not causal treatment effect.
- The scenario changes are illustrative and should not be treated as real customer interventions.
- The 12-month survival uplift values are modeled percentage-point differences, not realized retention.
- No booked revenue, ROI, defended LTV, or customer-lifetime-value outcome is claimed.
- Cross-validation and out-of-fold calibration do not replace external or temporal validation.
- Production use would require domain data, calibration, monitoring, model-risk governance, implementation controls, and stakeholder approval.
